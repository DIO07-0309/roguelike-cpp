# D1：Core Gameplay Loop（核心游戏循环设计）

> 基于当前 B8~B15 真实代码架构设计  
> 不推倒重来。以小步迭代方式演进。  
> 每个后续 D 阶段控制在 ~200~400 行代码。

---

## 一、当前状态分析

### 1.1 当前循环

```
进入楼层 → 探索房间 → 遭遇怪物 → 普攻/技能 → 击杀 → 掉落
  → 特殊房间(祭坛/宝箱/泉水) → 清空全部怪物 → 楼梯激活 → 下楼
  → 下一层重复（地图不同，怪物倍率不同，圣物刷新）
```

### 1.2 当前问题

| 维度 | 问题 |
|------|------|
| **楼层** | 1-4 层完全同质，只有 HP×/ATK× 倍率差异 |
| **战斗** | "走过去→普攻" 重复15层，没有节奏变化 |
| **怪物** | 所有怪物都是 chase→attack，B15 才有 Boss 技能 |
| **成长** | 只有数值：ATK+2、DEF+1、HP+10 |
| **剧情** | 零。只有 Boss 介绍画面 |
| **环境** | 零。只有地板和墙 |
| **资源** | 只有 HP 药水，没有管理空间 |
| **节奏** | 可以连续打 10 只怪然后下楼，没有呼吸感 |

### 1.3 已有资产（可直接利用）

| 系统 | 能力 |
|------|------|
| **MonsterType** (B14) | NORMAL/ARCHER/SHAMAN/BOMBER/TANK/ELITE，已有 6 种行为 |
| **BossState** (B15) | IDLE/ATTACK/CHARGE/SHOCKWAVE/SUMMON，已有技能循环 |
| **Buff** (B13) | poison/slow/attack_up，已有 DOT + STAT_MOD |
| **Relic** (B13) | 11 种，每层重新 Build |
| **SpecialRoom** (B8-10) | 祭坛/宝箱/泉水，E 键交互 |
| **Skill** (M6) | 4 主动 + 2 被动 + 3 级升级 |
| **DungeonGenerator** (M3) | BSP 随机地图，seed 驱动 |
| **DamageFloat/Shake/Freeze** (C1) | 伤害数字、屏幕震动、帧冻结 |

## 二、章节与楼层规划（5 章 × 3 层）

### 2.1 整体结构

```
章节 1 : 地牢入口 (Floor 1-3)
  主题：生存与基础
  环境：普通石砖地牢
  怪物：slime / orc / archer (独立行动)
  Boss：暗影骑士 (Floor 5)
  剧情：发现地牢的真相——这里曾是王城的地下监狱

章节 2 : 幽暗深渊 (Floor 6-8)
  主题：黑暗与视野
  环境：视野受限，需要灯火
  怪物：shaman 开始出现，给队友上 buff
  Boss：地狱火魔 (Floor 10)
  剧情：发现监狱深处囚禁着更可怕的东西

章节 3 : 熔岩炼狱 (Floor 11-13)
  主题：资源管理与站位
  环境：地面有灼热裂隙，踩上去掉血
  怪物：bomber/tank 开始出现，怪物开始配合
  Boss：深渊之主·终焉 (Floor 15)
  剧情：面对地牢的终极存在

过度章  : 旧日回廊 (Floor 4) / 深渊裂隙 (Floor 9) / 起源之门 (Floor 14)
  主题：喘息 + 大量资源 + 剧情推进
  环境：每章最后一层 (Boss 前) 是"休息层"
  特殊：多特殊房间、多掉落、环境叙事
```

### 2.2 每层设计

| 楼层 | 章节 | 新增机制 | 教学目的 | 环境特征 |
|------|------|----------|----------|----------|
| 1 | 入口 | 移动+攻击+拾取 | 基础操作 | 普通石砖 |
| 2 | 入口 | 特殊房间(祭坛) | Buff 机制 | 普通石砖 |
| 3 | 入口 | 特殊房间(宝箱) | 装备系统 | 普通石砖 |
| 4 | 回廊 | 大量宝箱+泉水 | 补资源,准备 Boss | 温暖火把 |
| **5** | **Boss** | **暗影骑士** | **Boss 机制** | **王座厅** |
| 6 | 深渊 | 视野受限(暗) | 站位意识 | 黑暗洞穴 |
| 7 | 深渊 | Shaman 出现 | 优先击杀辅助 | 黑暗洞穴 |
| 8 | 深渊 | 地刺陷阱 | 移动精度 | 尖刺大厅 |
| 9 | 裂隙 | 多泉水+宝箱 | 补资源,剧情揭示 | 荧光洞窟 |
| **10** | **Boss** | **地狱火魔** | **冲锋预警** | **熔岩王座** |

## 三、30 秒战斗节奏设计

### 3.1 节奏模板

```
[0-5s]   探索移动 —— 走向未探索区域
[5-10s]  发现敌人 —— 怪物进入视野，AI 开始反应
[10-15s] 战斗 —— 普攻 + 技能释放
[15-20s] 击杀/撤退 —— 收到掉落，评估血量
[20-25s] 拾取/恢复 —— 捡物品，喝药水，触发特殊房间
[25-30s] 继续探索 —— 走向下一个方向
```

### 3.2 节奏防止

| 反模式 | 预防 |
|--------|------|
| 连续打 3 只怪 | 把怪物间距控制在 8-12 格之间 |
| 一分钟无交互 | 每层至少 2 个特殊房间，保证 30s 内能遇到 |
| 打怪→捡→打怪 | 特殊房间穿插在怪物之间，打破重复 |

### 3.3 30 秒反馈机制清单

每 30 秒内，玩家至少触发一项：

- ☐ 发现新怪物类型
- ☐ 触发 Buff (poison/slow/attack_up)
- ☐ 进入特殊房间范围 (发现提示)
- ☐ 触发生效 Relic (吸血/图腾/毒牙)
- ☐ 捡到物品
- ☐ 升级
- ☐ 技能冷却完毕
- ☐ 发现楼梯
- ☐ 环境事件触发
- ☐ 剧情文字弹出

## 四、5 分钟成长路线

### 4.1 玩家成长里程碑

| 时间 | 层级 | 新增能力 | 非数值成长 |
|------|------|----------|------------|
| 0min | Lv1 | 移动 + 普攻 | 基础操作 |
| 1min | Lv1-2 | 习得第 1 个技能 | 新操作按键 |
| 2min | Lv2 | 捡到第 1 件装备 | 新伤害数字 |
| 3min | Lv2-3 | 触发第 1 个 Buff | 理解状态系统 |
| 4min | Lv3 | 遇到特殊房间 | 理解交互 |
| 5min | Lv3-4 | 技能升级 Lv2 | CD 变短 = 操作频率升 |
| 7min | Lv4 | 获得第 1 个 Relic | 新被动效果 |
| 10min | Lv5 | Boss 战 | 学习 Boss 机制 |
| 12min | Lv5-6 | 第 2 章开始 | 新环境+新怪物 |
| 15min | Lv6-7 | 第 2 个技能 | combo 组合 |
| 20min | Lv7-8 | Relic Build 成型 | 每层新 Build |
| 25min | Lv9 | Boss 10 | 更复杂机制 |
| 30min | Lv10+ | 第 3 章 | 综合挑战 |

### 4.2 非数值成长表

| 成长类型 | 实现方式 |
|----------|----------|
| 操作丰富 | 技能数量↑、CD 缩短、combo 出现 |
| 战术理解 | 怪物 AI 越来越复杂，需要学习克制 |
| 资源管理 | 后期药水数量固定，需要决策使用时机 |
| 风险评估 | Bomber 靠近→决策进攻或撤退 |
| 击杀顺序 | 多个敌人时优先打 Shaman 还是 Tank |
| 站位意识 | 地刺/裂缝/冲锋预警→必须走位 |

## 五、怪物成长路线

### 5.1 章节怪物行为演进

| 章节 | 怪物行为 | 具体表现 |
|------|----------|----------|
| 第1章 (F1-5) | **独立行动** | 每只怪单独行动，互不配合 |
| | Slime | chase→attack，25% 上 slow |
| | Orc | chase→attack，25% 上 poison |
| | Archer | 远程攻击，靠近则后退 |
| 第2章 (F6-10) | **开始配合** | Shaman 给队友 buff |
| | Shaman | 每 4s 给最近友方 attack_up 或 15% heal |
| | Shaman+Orc | 萨满 buff 兽人→兽人伤害大增 |
| 第3章 (F11-15) | **拥有技能/协同** | 高级怪物有特殊能力 |
| | Bomber | 自爆 AOE → 玩家必须走位 |
| | Tank | 高 HP + 慢速 → 挡路，掩护后方远程 |
| | Tank + Archer | 坦克挡路，弓箭手在后方输出 |
| | Shaman + Bomber | 萨满给炸弹怪回血→更难引爆 |
| | Elite | 出场带随机 Buff，属性强化 |

### 5.2 未来 D2/D3 扩展方向

| 阶段 | 怪物能力 | 代码改动量 |

## 八、后续 D2~D6 开发路线

### D2：Combat Evolution（战斗深度）~250 行

| 项目 | 说明 |
|------|------|
| **怪物技能** | 在 MonsterAI 的 `switch(MonsterType)` 中扩展：Orc 冲锋 / Slime 分裂 / Archer 散射 |
| **技能联动** | `CombatCoordinator` 中增加 combo 检测：斩击后接神罚=伤害+30% |
| **弱点系统** | Monster 加 `weak_to` 字段，特定攻击类型伤害×1.5 |

### D3：Enemy Cooperation（敌人协同）~300 行

| 项目 | 说明 |
|------|------|
| **Tank 嘲讽** | 坦克进入视野后玩家优先攻击 Tank（自动锁定） |
| **Archer 齐射** | 2+ 弓箭手同时存在时攻击力+30% |
| **Shaman 图腾** | 召唤一个不可移动的图腾，范围内所有怪物 attack_up |
| **双怪组合技** | Tank+Archer: 坦克冲撞击退玩家→弓箭手趁机输出 |

### D4：Floor Narrative（楼层环境）~350 行

| 项目 | 说明 |
|------|------|
| **环境系统** | `EnvironmentEffect` struct: 地刺/裂缝/毒雾/减速泥潭 |
| **视野系统** | 黑暗楼层：玩家视野限制 6 格，进入房间才照亮 |
| **剧情日志** | 特殊房间中发现"日志碎片"，拼凑故事 |
| **休息层** | Floor 4/9/14：怪物减少、特殊房间增加、环境温暖 |

### D5：Boss Evolution（Boss 机制战）~400 行

| 项目 | 说明 |
|------|------|
| **暗影骑士 2.0** | Phase3 召唤坐骑→不可攻击→先打坐骑→再打骑士 |
| **地狱火魔 2.0** | 定期制造熔岩区域→地板不能站→走位要求 |
| **深渊之主 2.0** | 最终形态：召唤所有已击败 Boss 的幻影 |
| **Boss 专属环境** | Boss 房间特殊地板/障碍物/互动物件 |

### D6：Endgame & Replayability（终局与重玩）~300 行

| 项目 | 说明 |
|------|------|
| **流派统计** | 通关后显示本局 Build 分析（主要伤害来源、使用最多的技能） |
| **周目+** | 通关后可选择 NG+，怪物+1 技能，但 Relic 池扩大 |
| **挑战模式** | 无 Relic / 无拾取 / 仅技能 等自选限制 |
| **种子分享** | 显示本局 seed，可输入种子重玩同一局 |

## 十、总结

D1 不是代码阶段，是设计阶段。

核心交付物是本设计文档。

后续 D2~D6 严格遵循本设计，每一步 ~200-400 行，在现有架构上增量演进。

整个游戏从 "堆砌系统" 转向 "设计循环"。
